WebFlux

WebFlux is Spring's reactive programming model that uses non-blocking I/O and event-loop concurrency. It's designed for high-concurrency, I/O-bound applications.

Run WebFlux Test
How WebFlux Works

WebFlux uses a reactive programming model based on the Reactor library. It uses a small number of event-loop threads to handle many concurrent connections. Instead of blocking threads, it uses callbacks and event handlers to process requests when data is available.

Characteristics:
  • Non-blocking I/O with event loops
  • Uses a small number of threads (typically CPU cores + 1)
  • Reactive programming model with Mono and Flux
  • Excellent for I/O-bound tasks with high concurrency
  • Requires a different programming model (reactive streams)
  • CPU-bound tasks need to be offloaded to avoid blocking event loops
