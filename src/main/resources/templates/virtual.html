Virtual Threads

Virtual threads are lightweight threads that don't map 1:1 to OS threads. They're designed for I/O-bound workloads with high concurrency.

How Virtual Threads Work

Virtual threads (Project Loom) are a feature introduced in Java 21 that allows for a many-to-few mapping between virtual threads and OS threads. This means you can create millions of virtual threads without exhausting system resources.

Characteristics:
  • Many-to-few mapping between virtual threads and OS threads
  • Extremely lightweight (a few KB of memory per thread)
  • Can create millions of virtual threads
  • Automatically yield during blocking operations
  • Excellent for I/O-bound tasks with high concurrency
  • No special programming model required (use standard Java concurrency)
